1. 맵 순서 관리
반드시 진행 순서대로 배치.
포함관계 잘 이용할 것.

순서 바꾸는 방법
[
	MV 에디터 상 좌측 하단에서 해당 맵을
	드래그 해서 원하는 위치에 배치.
	이 때 불가피하게 포함 관계가 생김.
	이것을 ./data/MapInfos.json 에서
	"parentId" 항목 값을 수정해 해결.
	( 에디터 재시작해야 반영 )

	즉, 순서만 먼저 맞추고 -> 포함 관계 재정립.
]


2. Parallax Mapping 금지
( Parallax Mapping = 
이미지 한장을 통째로 타일셋화 시켜 사용하는 기술 )
플러그인 버그, 메모리 누수 문제로 금지.
Legacy 타일셋 시스템만 활용해야.


3. BGM 페이드, loop 고려
페이드 인, 페이드 아웃이 원칙.
오래 쓰일 BGM은 자연스러운 loop 고려.
Loop은 크로스페이드, 페이드 인아웃 사용할 것.


4. 스크립트 편집시 superclass는 가능한 한 손대지 말 것
superclass를 편집하면 관련한 모든 기능이
원치 않는 방향으로 영향받을 수 있으니 주의.


5. (1.)의 경우 제외하면 JSON 파일 직접 편집은 자제
프로젝트를 저장할 때,
JSON 파일 내용이 덮어씌워지므로 변경 사항이 매번 초기화.
따라서 의미가 없음.


6. 그림 번호 규칙
그림 표시 기능에서 숫자가 큰 그림이 위에 그려짐.
따라서 캐릭터 일러스트는 2 이상의 번호를 사용.
( 1은 배경으로 Reserve )


7. 스위치 번호
스위치는 20개씩 한 페이지인데,
맵 구간별로 스위치 페이지를 할당.
(ex. 1페이지는 사고현장~병원 입장 전)
미사용 스위치가 있더라도 용량문제는 없으니,
페이지를 지키는 편이 효율적.


8. 스위치/변수 이름
유형 + 상세장소 + 설명 으로 구성.
유형과 상세장소는 생략가능, 권장하지는 않음.
- 유형은
(1) 그래픽적 요소가 주가 되는 변화는 vis_
(2) 사운드 관련 요소는 sfx_ 를 사용.
(3) 기타 유형을 표시할 수 있는 접두어를 사용가능
- 상세장소는
(1) C1F, D2F, C402, D501, Xray, DocsPed 등 과 같이 표시.
(2) 별다른 의미가 없는 Room, Office 등은 전부 생략.
- 설명은
(1) 이름이 길더라도 단번에 이해되게.
C5F_Closet 보다는 501_ClosetUseItemVar이 이해하기 쉬울 것.
(2) 단어마다 대문자를 적절히 사용할 것.
readextrtext 보다는 readExtrText가 보기 쉬울 것.


9. 셀프스위치의 예외적 허용
일회성 장치에만 : 점프스케어(깜놀) 등.
이벤트 페이지가 너무 길어질 때.


10. 공통 이벤트
반드시 필요한 경우가 아니라면,
발동 조건과 발동 스위치 조건을 '없음'으로 해둘 것.


11. 아이템
기본값은
아이템 유형	:	상비 / 핵심
소모품		:	하지 않는다
범위		:	없음
사용 가능할 때	:	항상 / 사용불가
아이템 분류 유형에 따라 '상비, 핵심'
메뉴에서 사용해 (적힌 글자를 읽는 등) 동작시킬 수 있다면 '항상', 외에는 '사용불가'
실수하기 싫다면 반드시 원래 있던 항목을 복사해서 사용.
사용 효과에는 최대 하나의 공통이벤트만 들어갈 것. (어차피 하나만 작동하므로)

12. 맵 Notification
맵 상에서 특별한 장소의 경우 '장소'
맵 상에서 각층에서 중복된 장소의 경우 '층 + 장소'
맵 상에서 해당 층에 다수의 중복된 장소가 있는 경우 '층 + 제n + 장소'
